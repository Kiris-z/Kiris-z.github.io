1.Windows 下 vscode 调试 C++ 代码，STL 只显示地址不显示数据

# bug 复现

2023 年 4 月。

当在 C++ 代码中使用 STL 容器，调试过程中在左侧监视窗口查看 STL 内容，只会显示地址，而非 STL 容器内的值。该问题出现在将旧版本的 vscode 更新为 1.77.3 版本后发生，切换为 1.69 版本后会恢复正常。当第一次切换为某些 1.7 几版本的时候，会短暂正常，在一到两小时后，该 bug 就会出现，诶，这一点就很神奇。

# 解决过程

看网友之前的帖子，发现对该 bug，有两种看法。

一种是没有开启 -enable-pretty-printing ，个人认为这种情况的可能性还是比较低的，毕竟在你配置环境的时候，就自动为你配置了该项。

另一种就是版本的问题了，我尝试手动更换各种版本的 mingw，采用 LLVM+clang 编译均无法解决该 bug。

最后在官网发现配置 C++ 环境的教程已经改为使用 MSYS2 来安装 mingw64。通过 pacman -S --needed base-devel mingw-w64-x86_64-toolchain 命令来安装最新版本的 mingw64，可以完美解决该 bug。

教程如下：

https://code.visualstudio.com/docs/cpp/config-mingw

# 注意点

在安装配置 VScode C++ 环境之前，一定要将之前的 mingw64 及其环境变量删除干净，并重启电脑。

在 mysys2 中使用 pacman -S --needed base-devel mingw-w64-x86_64-toolchain 时，可能会出现镜像拉取失败的情况，多运行几遍该命令，直到全部安装成功即可。
