# 概念

408 中，整数补码，移码符号位相反，数值位相同

双符号位的最高符号位代表真正的符号，而低位符号位用于参与移位操作以判断是否发生溢出

C 语言的数据在内存中为补码表示形式

补码减法或是无符号数减法时，需计算减数负数的补码，通过一个反向器和控制端 Sub 实现取反，同时将低位进位置一来实现 + 1 操作

# 补码双符号位规格化

  1. 当 10.XXX或01.XXX10.XXX或01.XXX10.XXX或01.XXX 需要右规一次
  2. 当 00.0XXX或11.1XXX00.0XXX或11.1XXX00.0XXX或11.1XXX 需要左规一次或多次
  3. 当 00.1XXX或11.0XXX00.1XXX或11.0XXX00.1XXX或11.0XXX 规格化完成

# 溢出判断

  1. 一位符号位，参与操作数的符号相同，同时与结果的符号不同，即 V=AsBsSs‾+As‾Bs‾SsV=A_sB_s\overline{S_s}+\overline{A_s}\overline{B_s}S_sV=As​Bs​Ss​​+As​​Bs​​Ss​。
  2. 双符号位，略
  3. 符号位进位与最高数位进位不同，即 V=Cs⊕C1V=C_s \oplus C_1V=Cs​⊕C1​。

# 进位借位标志 CF

小的减大的，借位

相加溢出，进位

即 CF=Cout⊕CinCF=C_{out}\oplus C_{in}CF=Cout​⊕Cin​。

# 移位操作

# 算数移位

符号位不变，其余位置移位

负数反码补 1，负数补码右移补 1，其余补 0

# 逻辑移位

补 0

# 循环移位

  1. 带标志位
    标志位增加到最高位前参与移位

  2. 不带标志位

    右移，高位进入标志寄存器和低位

    左移，低位进入标志寄存器和高位

# IEEE 754 标准

浮点数真值为

(-1)^S\times 1.M\times2^{E-bias}(bias=127\text{或}1023) $$。 ### 阶码E 真值范围为 -126~127 移码表示，字面值范围为 1~254 这是由于阶码(移码)全0和全1时，有特别解释。而加上偏置值127后移码变为全0或全1的数为-127和128(-0)，由于符号位的出现，128即为-0转化为移码后与+0相同。而-127则舍去作为特别定义。 ### 尾数M 有1位隐藏位，剩余位范围为 $00\cdots 0-11\cdots 1$ 。 真值范围为 $2-2^{-0}\sim2-2^{-23}$。 # 定点数乘除运算 ## 原码一位乘 n 次移位，n 次加法运算 ## 补码一位乘(Booth算法) n 次移位，n+1 次加法运算 ## 原码一位除(不恢复余数法) ## 补码一位除(加减交替法) # 题型 ## 负数补码大小比较

[x_\text {补}] = 1,x_1x_2x_3x_4x_5x_6

负数真值越大，即真值越接近 0，即真值的绝对值越小 则 $x_1x_2x_3x_4x_5x_6$ 字面值越大
